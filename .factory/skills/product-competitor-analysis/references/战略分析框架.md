- 产品名称：

- 场景（可选：ToC/ToB/平台能力/生态内功能）：

【输出格式（严格按顺序）】

0) 一句话结论（先站队）
- “战略价值：高/中/低”，并用一句话说透原因。

1) 战略角色定位（必须给出）

- 如果它不直接赚钱，它在提升哪一个核心系统指标？（必须可量化）

2) 核心问题（Key Problem）
- 它真正解决的“关键问题”是什么？（用用户语言描述）
- 需求强度×频率×支付能力：各给一个判断（高/中/低）+理由
- 替代方案是什么？用户为什么不用替代方案？

3) 决策机制：理性还是身份/情感？
- 用户选择它主要是：理性ROI / 情感爽感 / 身份认同 / 社交货币（可多选）

4) 增长：靠渠道还是靠结构？
- 主要增长引擎是：渠道驱动 / 结构内置 / 混合（选1）
- 写清机制因果链：A → B → C（例如：作品可分享→传播→自然新增）
- 最大增长瓶颈在 AARRR 哪一环？（选一个）

5) 留存：习惯 / 成本 / 网络效应（护城河形态）
- 留存主因属于哪类：习惯（高频）/ 切换成本（资产沉淀）/ 网络效应（越用越强）
- 对应的“留存资产”是什么？（模板/项目/数据/关系链/工作流/集成等）
- 最容易塌的留存点是什么？

6) 商业化：目的还是副产物？
- 商业化路径：订阅/用量/席位/抽成/增值服务/带动主业务（选1-2）
- 为什么这个计费单位合理？（价值锚点 + 成本结构）
- 单位经济学粗判断：毛利压力点在哪里？（推理成本/交付成本/获客成本/客服合规）

7) 抗压性测试：资源减半还能成立吗？
- 资源减半时，最先砍什么？必须保什么？（给出“保留清单”）
- 说明：如果只能做1条增长引擎，它会选哪条，为什么？

8) 真护城河 vs 阶段红利
- 不可复制优势（列3条，按强弱排序）
- 阶段红利/容易被抄的部分（列3条）
- 给出“12-24个月后仍成立”的前提条件（列2条）

9) 逆向建议：下一步千万别走哪条路
- 列出1-3条“最危险的诱惑”（例如：过早平台化、乱扩品类、过度投放、过早企业化等）
- 每条写清：为什么会死 + 预警信号（看到什么指标/现象就要停）

10) 90天验证与资源决策（必须给出）
- 北极星指标（NSM）1个 + 输入指标3个
- 90天里程碑：达标阈值是什么？
- 决策规则：达标→加码做什么；不达标→止损/转向做什么

【硬性要求】
- 每个判断尽量落到“机制 + 指标 + 可验证方式”
- 信息不足允许合理假设，但必须标注【假设】并写【验证方式】
