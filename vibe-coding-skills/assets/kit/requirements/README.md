# requirements/

本目录用于按 feature 沉淀业务产物，建议结构：

```
requirements/<feature>/
  spec.md      # 目标/非目标/验收
  ux.md        # 交互与状态流（如适用）
  design.md    # 技术方案
  review.md    # 评审清单与签字
  tasks.md     # 可执行任务拆解 + 自动化验收
  status.md    # 当前阶段与任务推进、验收记录
  changes.md   # 范围/设计变更记录
```

执行规则以仓库根目录 `AGENTS.md` 为准。
